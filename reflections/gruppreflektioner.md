# Gruppreflektioner

Svara på var och en av frågorna nedan som grupp. MINST 100 tecken per fråga.

## Frågor

### Inledning

#### Sammanfatta projektets syfte, samt era gemensamma mål

Projektets syfte var att vi dels skulle jobba agilt och sen även få till en sida enligt en figma-skiss.
Våra gemensamma mål var att leverera alla sidor i tid, följa kravlistan(se nedan),
jobba med GitHub issues/kanban och få till en responsiv webbplats.

Skapa och designa en hemsida för “Bonzai” enligt kundens önskemål.
Responsiv webbsida.
Semantiskt korrekt skriven kod och struktur.
Lätt för användaren att hitta lediga rum efter sina specifika önskemål och information.
Säkerhetsställa att hemsidan fungerar på alla enheter som finns på marknaden.
Sidan “Bonzai” utstrålar AI.

#### Hur upplevde teamet projektet initialt?

Det var ett roligt och utmanande projektarbete med HTML, CSS och slutligen JavaScript.
Det agila var utmanande, och att dessutom få kläm på det samtidigt som man jobbar med koden var utmanande. Teamet upplevde att det blev tydligare när vi delade upp sidorna, satte upp hela Kanban board och började jobba i små steg.

### Planering och genomförande

#### Beskriv hur ni inom teamet planerade projektet initialt

Vi började projektet med att gemensamt gå igenom projektets uppgiftsbeskrivning, skissen och alla olika krav. För att strukturera upp arbetet använde vi oss utav en Kanban-board på GitHub, där skapade vi olika kort med user stories, acceptanskriterier och estiminering-nivå. Sedan bröt vi ut korten till tasks och vi kom överens om att vi skulle göra varsin sida, så alla kort som tillhörde sid 1 assigna Fideli till sig, sid 2 assigna Jimmy till sig och sid 3 assigna Linda till sig. Vi la till en extra kolumn som hette sprint backlog, den uppdaterade vi varje vecka med vad fokuset skulle ligga på.

Vi bestämde även riktlinjer för kodstruktur och design för att säkerställa att kunden blev nöjd.

#### Vilka steg har teamet tagit för att utveckla applikationen?

Utvecklingen av applikationen skedde stegvis. Vi började med korten som hade högst prioritet och där kom vi gemensamt över att det var en del på varje sida som var extra viktig. På sid 1 var det check in/out, på sid 2 var det samma check in/out och på sid 3 var det world class amenities. Men även uppdelningen av root och gemensam css-kod, footer och header. Vi hade daily stand-up och visade hur vi låg till, samt varje fredag hade vi sprint review med PO. Så stegvis varje vecka har vi betat av kort efter kort och med en tidsgräns gick vi tillslut i mål.

### Teamets utmaningar och lösningar

#### Vika var de största utmaningarna för teamet?

De största utmaningarna var att få sidorna att se bra ut, samt att när flera personer kodar kan olika merge-konflikter kan uppstå. Responsiviteten var även klurig att få till på alla sektioner, men även rätt semantik och välstrukturerad css fil ställde till det ibland.

#### Hur löste eller hanterade teamet dessa utmaningar?

Strukturmässigt använde vi oss utav en gemensam root-variablier, wrapper och kollade igenom varandras koder innan vi mergea. Vi använde olika resurser för att lösa olika GitHub-konflikter, alltifrån google/ChatGPT till GitHub’s egna “fusklapp”.

#### Vilka kompromisser inom teamet har ni gjort under projektets gång?

Vi kompromissade kring olika detaljer i designen med tidspress, exempelvis font-storlekar och spacing. Vi utgick från figma-skissen så gott som vi kunde, men när förändringen kom var vi alla tvungna att tänka om och kompromissa rejält.

### Teamets reflektion och utvärdering

#### Vad lärde sig gruppen inom projektet?

Vi fick dels lära oss HTML, CSS och Java Script men även lära oss att jobba agilt och få en mer djupdykning i GitHub. Vi jobbade med en Kanban board och fick lära oss mycket om user stories, estimering, tasks, daily stand up och även sprint review. Vi har lärt oss mycket under kort tid!

#### Hur kan gruppen förbättra sitt samarbete framöver?

Vi kan förbättra samarbetet genom att använda mer av samma kod. Eftersom att vi delade upp arbetet med varsin sida så hade man inte helt full koll på varandras kod, vi hade kunnat förenkla koden mer om vi tex använde mer gemensamma variabler.
